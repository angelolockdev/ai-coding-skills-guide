01 - Le kit

Chaque skill a un rôle précis.

Le piège n'est pas de manquer d'outils. C'est de demander à un seul prompt de définir le besoin, produire le code, juger le design et valider le résultat.

01

Grill me

Force les décisions produit avant la première ligne de code.

À utiliser quandLe besoin est vague, coûteux ou irréversible.
npx skills@latest add mattpocock/skills --skill grill-me
02

APEX

Analyse, planifie, exécute et examine avec une preuve finale.

À utiliser quandLa fonctionnalité est nette et doit être testée.
npx skills@latest add melvynx/aiblueprint --skill apex
03

Impeccable

Donne une direction visuelle intentionnelle aux écrans trop génériques.

À utiliser quandL'UI fonctionne, mais ne dit rien du produit.
npx impeccable install
04

Feel Better

Corrige les détails qui séparent une UI correcte d'une UI soignée.

À utiliser quandIl reste des frictions de rythme, contraste ou interaction.
npx skills add jakubkrehel/make-interfaces-feel-better
05

Thermo-Nuclear

Met le code sous pression avant qu'il ne parte en production.

À utiliser quandIl faut évaluer dette, sécurité et maintenabilité.
npx skills add https://github.com/cursor/plugins --skill thermo-nuclear-code-quality-review
02 - Astuces concrètes

Les détails qui augmentent vraiment le niveau.

Ces règles réduisent les faux départs, évitent le code jetable et font de l'IA un partenaire de production plutôt qu'une machine à générer des hypothèses.

01

Verrouillez le “done”

Avant toute implémentation, faites écrire à l'agent les critères observables : état vide, erreur, mobile, accessibilité, test et preuve attendue.

02

Un écran, une intention

Commencez une itération UI avec une phrase : “Cette page doit faire ressentir / permettre…”. Impeccable aura une direction, pas un vide à remplir.

03

Demandez les preuves

Un “c'est fait” ne suffit pas. Demandez les commandes exécutées, leur sortie utile, les scénarios testés et, pour une UI, une capture.

04

Réduisez le rayon d'action

Donnez une tâche verticalisée. Mieux vaut une petite tranche utilisable qu'une refonte “globale” qui mélange architecture, design et dette.

05

Conservez les décisions

Faites résumer les hypothèses et arbitrages dans AGENTS.md, une spec ou le PR. Le prochain agent ne devra pas les réinventer.

06

Auditez à froid

Le générateur ne doit pas être son unique juge. Terminez par une revue séparée, agressive sur les cas limites, les simplifications et les secrets.

Un workflow qui ne saute pas d'étape.

Chaque phase produit un artefact vérifiable. La phase suivante ne doit jamais dépendre d'une supposition de l'agent précédent.

05 - FAQ

Les questions à se poser avant de lancer un agent.

Une bonne demande donne du contexte, des contraintes et une définition claire de ce qui doit être vérifié.

Faut-il installer les cinq skills ?

Oui si vous voulez le workflow complet. Sinon, commencez par Grill me et APEX : ils éliminent beaucoup de rework avant même la question visuelle.

Est-ce que cela remplace une revue humaine ?

Non. Ces skills structurent et renforcent la revue. Toute modification sensible, publication ou action externe garde une validation humaine.

Quel prompt donne les meilleurs résultats ?

Un prompt qui indique le but, le périmètre, les contraintes, les fichiers pertinents, le résultat attendu et comment le vérifier.

Pourquoi séparer Impeccable et Feel Better ?

Impeccable définit l'intention et le système visuel. Feel Better améliore les détails d'exécution : surfaces, ergonomie, motion et stabilité.

Ne demandez plus à l'IA de “faire une app”.

Donnez-lui une étape précise, un niveau d'exigence et la preuve à produire. Le résultat devient plus rapide à relire, plus simple à maintenir et plus fiable à livrer.
